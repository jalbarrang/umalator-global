# Create Cursor Command

Create a new, concise Cursor Command that follows project conventions.

## Key Principles

Commands should be:

- **Focused** - One clear task or workflow
- **Concise** - Under 50 lines, ideally under 30
- **Actionable** - Steps, not reference material
- **Scannable** - Clear structure with headers

## Template Structure

```markdown
# [Command Name]

Brief overview (1-2 sentences of what this command does).

## Steps

1. **Action 1**
   - Key details
   - Specific guidance
2. **Action 2**
   - Key details
3. **Action 3**
   - Key details

## [Command Name] Checklist

- [ ] Clear, specific outcome 1
- [ ] Clear, specific outcome 2
- [ ] Clear, specific outcome 3
```

## Anti-patterns to Avoid

- Reference documentation (link to docs instead)
- Large code templates or examples (>20 lines)
- Persona instructions with extensive context
- Multiple unrelated tasks in one command
- Aspirational checklists with >10 items

## Command Creation Checklist

- [ ] Single, well-defined task
- [ ] Under 50 lines total
- [ ] 2-4 clear steps
- [ ] Actionable checklist (3-6 items)
- [ ] No embedded templates or reference material
